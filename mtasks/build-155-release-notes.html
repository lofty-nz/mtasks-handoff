🏗 Recent build notes

b155 — 📸 Screenshot-to-clipboard button (#1198). There's a new 📸 button in the top-right toolbar (just after 🔍). Tap it to capture the current view and copy it to the clipboard as a PNG — then paste it straight into a chat, a note, anywhere. It's built on the browser's native Screen Capture + Clipboard APIs, so no libraries and nothing added to the single-file build. The first tap shows the browser's quick “share this tab” confirm (it's pinned to the current tab, so it's one tap), then it grabs a frame and copies it. One requirement: it only works over a secure context — your Tailscale HTTPS address and localhost both qualify; a plain http:// LAN address won't (the browser blocks screen capture there), and you'll get a clear message if so.
b154 — 🩺 Setup health monitor (#1197) + readable, collapsible build history (#1196). Needs the new server.py deployed (for the health checks). #1197: a new Setup health panel on the Dev page (just under the pipeline) watches the fragile parts that keep everything alive — app server, data file, Tailscale, GitHub CLI and the git remote — each with a green / amber / red dot and a one-line detail, plus an overall all ok / N warn / N down summary so a problem jumps out. It checks once when you open Dev and on 🔄 Refresh; the server side is a new /health endpoint that runs tailscale status, gh auth status etc. (Works over the local server only — file:// can't reach it.) #1196: the Build history list in Version history is no longer a wall of escaped <b> tags — each build is now a collapsible row showing its title, expanding to the full, properly-formatted note.
